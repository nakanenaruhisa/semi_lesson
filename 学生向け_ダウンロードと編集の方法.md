# 初学者向け：GitHub から教材をダウンロードして編集する方法

この教材（semi_lesson）は GitHub にあります。**自分のパソコンに取り込んで、自由に編集・実行**できます。  
GitHub のアカウントはなくても大丈夫です。

---

## ⚠ 必ず守ること：教材の保存場所

**ダウンロード・解凍した教材は、必ず次の場所に置いてください。** 展開場所を守らないと、授業での説明やパスの例と合わず、トラブルのもとになります。

### Mac の場合（推奨パス）

```
/Users/[あなたのユーザー名]/Library/CloudStorage/OneDrive-学校法人立命館/lecture/semi/R/semi_lesson
```

- **`[あなたのユーザー名]` の部分は、パソコンごとに違います。**
- 例に書いている **`naruhisa007`** は**教員のユーザー名**です。あなたの Mac のログイン名（ユーザー名）に置きかえてください。
- 自分のユーザー名の確認方法：**システム設定 → ユーザとグループ**、またはターミナルで `whoami` と入力して Enter。

**具体例**：ユーザー名が `tanaka` の場合は  
`/Users/tanaka/Library/CloudStorage/OneDrive-学校法人立命館/lecture/semi/R/semi_lesson`  
に `semi_lesson` フォルダ（中に Lesson01 などが入っている状態）がくるようにしてください。

### やることの整理

1. 上記のパスで、**`semi_lesson` という名前のフォルダ**ができるようにする。
2. ZIP の場合は、ZIP を**このパスの直下に解凍**するか、解凍して出てきた `semi_lesson` フォルダを**このパスに移動**する。
3. Git の場合は、**このパスのひとつ上のフォルダ**（`R` のなか）で `git clone` を実行し、`semi_lesson` がこのパスにできるようにする。

**展開場所を徹底してください。** 指定したパス以外（デスクトップや書類フォルダなど）に置かないでください。

- **デスクトップには、教材や作業用のファイル・フォルダを置かないでください。** 授業で指導している運用ルールです。デスクトップはあくまで一時的な表示用とし、保存場所は必ず上記のパスにしてください。

- **Windows** を使っている場合：ドライブや OneDrive の場所が異なります。授業での指示か、同じフォルダ構造（`…\OneDrive-学校法人立命館\lecture\semi\R\semi_lesson`）になるように保存場所を合わせてください。

## おすすめの2つの方法

### 方法1：ZIP でダウンロード（いちばん簡単・おすすめ）

**向いている人**：まず触ってみたい、Git を入れていない、更新はあまり取らなくてよい。

1. ブラウザで次のページを開く：  
   **https://github.com/nakanenaruhisa/semi_lesson**
2. 緑色の **「Code」** ボタンをクリックする。
3. **「Download ZIP」** をクリックする。
4. ダウンロードした ZIP を**解凍**する（ダブルクリックや右クリック→「展開」など）。
5. 出てきた **`semi_lesson` フォルダ**を、**必ず守ること**で指定した保存場所（`/Users/[あなたのユーザー名]/Library/CloudStorage/OneDrive-学校法人立命館/lecture/semi/R/semi_lesson`）に移動する。展開場所を守ること。
6. そのフォルダを **RStudio** や **Cursor** で開き、中の R ファイルを自由に編集・実行する。

**注意**：先生が教材を更新したときに「最新版」を取りたい場合は、もう一度 ZIP をダウンロードして、必要なフォルダだけ上書きするか、新しいフォルダに解凍して使ってください。

---

### 方法2：Git でクローン（更新を取りやすい）

**向いている人**：先生の更新を簡単に取り込みたい、少しずつ Git に慣れたい。

#### 事前準備：Git を入れる

- **Windows**：[Git for Windows](https://git-scm.com/download/win) をダウンロードしてインストール。
- **Mac**：ターミナルで `xcode-select --install` を実行するか、[Git for Mac](https://git-scm.com/download/mac) からインストール。

#### 手順

1. **教材を置く場所は「必ず守ること」のパスにすること。** まず、`/Users/[あなたのユーザー名]/Library/CloudStorage/OneDrive-学校法人立命館/lecture/semi/R` までフォルダを作成し、その **`R` フォルダ**に移動する（ターミナルで `cd` するか、Finder でそのフォルダを開く）。
2. その **`R` フォルダの中**で **ターミナル** を開く。
3. 次のコマンドを **1行** 入力して Enter する。

   ```bash
   git clone https://github.com/nakanenaruhisa/semi_lesson.git
   ```

4. 同じ場所に **`semi_lesson` フォルダ**ができる（＝指定の保存場所になる）。その中で R や RStudio / Cursor を使って編集・実行する。

**先生が教材を更新したとき**：`semi_lesson` フォルダでターミナルを開き、次を実行すると最新版が取り込めます。

```bash
git pull
```

自分で編集したファイルと競合した場合は、別のフォルダに新しく `git clone` し直して「最新版」として使う方法が簡単です。

---

## 共通のルール

- **教材の保存場所は「必ず守ること」のパスにすること。デスクトップにはファイル・フォルダを置かない**（授業で指導している運用ルールです）。
- **GitHub 上の教材は編集しない**（あなたには書き込み権限がないので、そのままでは変更できません）。
- 教材は**自分のパソコン（ローカル）**だけで自由に編集・実行してよい。
- 編集した内容を GitHub に戻す（push）必要はありません。レポート提出などは授業の指示に従ってください。

---

## まとめ

| 方法 | メリット | デメリット |
|------|----------|------------|
| **ZIP でダウンロード** | 手軽・Git 不要 | 更新時は再度 ZIP を取る必要あり |
| **Git でクローン** | 更新を `git pull` で取りやすい | Git のインストールが必要 |

**まずは「ZIP でダウンロード」から始めて、更新をよく取りたい人だけ Git を試す**のがおすすめです。
