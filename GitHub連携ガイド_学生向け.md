# ゼミ教材のダウンロード・利用マニュアル（学生向け）

この教材（`semi_lesson`）は GitHub で公開しています。**自分のパソコンにダウンロードして、自由に編集・実行**できます。GitHub のアカウントがなくても大丈夫です。

> **大前提**
> - 教材は GitHub 上で「読み取り専用」です。皆さんが GitHub 上のファイルを変更することはありません。
> - ダウンロードした教材は、自分のパソコン上で **自由に編集・実行** してかまいません。
> - 編集した内容を GitHub に戻す（push する）必要はありません。レポート提出などは授業の指示に従ってください。

ブラウザでアクセスすると、Lesson ごとのフォルダや R スクリプト、サンプルデータなどが確認できます。

## 2. 保存場所のルール（必ず守ること）

**ダウンロード・クローンした教材は、必ず以下の場所に保存してください。** 場所を守らないと、授業での説明やパスの例と合わず、トラブルのもとになります。

### Mac の場合

```
/Users/[あなたのユーザー名]/Library/CloudStorage/OneDrive-学校法人立命館/lecture/semi/R/semi_lesson
```

- **`[あなたのユーザー名]`** の部分はパソコンごとに異なります。自分のユーザー名は、**システム設定 → ユーザとグループ** で確認するか、ターミナルで `whoami` と入力して Enter してください。
- 例：ユーザー名が `tanaka` の場合 → `/Users/tanaka/Library/CloudStorage/OneDrive-学校法人立命館/lecture/semi/R/semi_lesson`

### Windows の場合

ドライブや OneDrive の場所が異なります。同じフォルダ構造（`…\OneDrive-学校法人立命館\lecture\semi\R\semi_lesson`）になるように保存場所を合わせてください。授業での指示にも従ってください。

### 注意事項

- **デスクトップには教材や作業用のファイル・フォルダを置かないでください。** これは授業で指導している運用ルールです。デスクトップはあくまで一時的な表示用とし、保存場所は必ず上記のパスにしてください。
- 最終的に `semi_lesson` という名前のフォルダ（中に `Lesson01_R_basics` などのレッスン用フォルダが入っている状態）ができていれば OK です。

## 3. 教材をダウンロードする（2つの方法）

### 方法 A：ZIP でダウンロード（いちばん簡単・おすすめ）

**向いている人**：まず触ってみたい、Git を入れていない、更新はあまり取らなくてよい。

1. ブラウザで **https://github.com/nakanenaruhisa/semi_lesson** を開く。
2. 緑色の **「Code」ボタン** をクリック → **「Download ZIP」** を選択。
3. ダウンロードされた ZIP ファイルを解凍する（ダブルクリックや右クリック →「展開」など）。
4. 解凍してできたフォルダを、**セクション 2 で指定した保存場所**に移動する。フォルダ名が `semi_lesson-main` になっている場合は `semi_lesson` にリネームしてください。
5. そのフォルダを RStudio や Cursor で開き、中の R ファイルを自由に編集・実行する。

> **教材が更新されたら？**
> もう一度 GitHub から ZIP をダウンロードし、必要なフォルダだけ上書きするか、新しいフォルダに解凍して使ってください。自分で編集したファイルを残したい場合は、古いフォルダを別名で保管しておくと安心です。

---

### 方法 B：Git clone でダウンロード（更新を取りやすい）

**向いている人**：先生の更新を簡単に取り込みたい、少しずつ Git に慣れたい。

#### ステップ 1：Git のインストール

| OS | インストール方法 |
|----|------------------|
| **Windows** | [Git for Windows](https://git-scm.com/download/win) をダウンロードしてインストール |
| **Mac** | ターミナルで `xcode-select --install` を実行、または [Git for Mac](https://git-scm.com/download/mac) からインストール |

#### ステップ 2：教材をクローン（コピー）する

ターミナル（Windows は Git Bash、Mac はターミナル.app）を開き、**保存場所のひとつ上のフォルダ（`R` フォルダ）に移動してから** 次のコマンドを実行します。

```bash
cd "/Users/[あなたのユーザー名]/Library/CloudStorage/OneDrive-学校法人立命館/lecture/semi/R"
git clone https://github.com/nakanenaruhisa/semi_lesson.git
```

実行すると `R` フォルダの中に `semi_lesson` フォルダが作られ、教材一式が入ります。これでセクション 2 で指定した保存場所に配置されます。

#### ステップ 3：教材を開いて学習する

`semi_lesson` フォルダを RStudio や Cursor で開き、各 Lesson の R スクリプトを実行してください。

> **教材が更新されたら？**
> `semi_lesson` フォルダの中でターミナルを開き、次のコマンドを実行すると最新の教材を取得できます。
>
> ```bash
> git pull
> ```
>
> もし自分の編集とぶつかってエラーが出た場合は、別のフォルダに新しくクローンし直すのが確実です。

## 5. よくある質問（FAQ）

### Q1. GitHub のアカウントは必要ですか？

**不要です。** ZIP ダウンロードでも Git clone でも、公開リポジトリなのでアカウントなしで取得できます。

### Q2. 教材を自分で書き換えても大丈夫ですか？

はい。ダウンロードした教材は自分のパソコン上のコピーなので、**自由に編集してかまいません**。GitHub 上の原本には影響しません。

### Q3. 教材が更新されたかどうか、どうやって分かりますか？

授業やチャットで先生からアナウンスがあります。また、GitHub のページにアクセスすると、各ファイルの最終更新日時が確認できます。

### Q4. `git pull` したらエラーが出ました

自分が編集したファイルと先生が更新したファイルが同じだった場合に競合（コンフリクト）が起きることがあります。一番かんたんな解決方法は、**別のフォルダに新しくクローンし直す** ことです。

```bash
git clone https://github.com/nakanenaruhisa/semi_lesson.git semi_lesson_new
```

これで `semi_lesson_new` というフォルダに最新版が入ります。自分の編集は元のフォルダに残っているので、必要な部分だけコピーしてください。

### Q5. Mac で「git がインストールされていません」と出ます

ターミナルで以下を実行し、Xcode コマンドラインツールをインストールしてください。

```bash
xcode-select --install
```

ダイアログが表示されたら「インストール」をクリックして完了を待ちます。

### Q6. 自分のユーザー名が分かりません

ターミナルで次のコマンドを実行してください。表示された文字列があなたのユーザー名です。

```bash
whoami
```

## まとめ

| 方法 | メリット | デメリット |
|------|----------|------------|
| **ZIP でダウンロード** | 手軽・Git 不要 | 更新時は再度 ZIP を取る必要あり |
| **Git でクローン** | 更新を `git pull` で取りやすい | Git のインストールが必要 |

**まずは「ZIP でダウンロード」から始めて、更新をよく取りたい人だけ Git を試す**のがおすすめです。
